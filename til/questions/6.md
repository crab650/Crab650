## 問題

外銷客戶都被設為零稅 或免稅 ，外銷客戶都是商業發票
發票還有一種是人工選號 ， 及 手寫發票
台灣法律規定 發票需依時間規定開立 ， 第一種方法是電腦依序給號
有些客戶是希望能開先前日期的發票 ， 例如本月出貨希望開成上月底出貨 ， 因此需預留發票號碼 ， 供日後不時之需 ， 此時用人工選號

# 一、外銷客戶為什麼都是零稅率？

很多 ERP 會將：

```
外銷客戶
```

設定為：

```
VAT Rate = 0%
```

或：

```
Zero Rated
```

原因：

不是：

不用開發票。

而是：

依照台灣《營業稅法》，

**出口貨物適用零稅率（0% VAT）。**

---

## 什麼叫零稅率？

很多人容易和：

```
免稅
```

搞混。

其實：

完全不同。

---

零稅率（Zero Rated）

表示：

```
稅率 = 0%
```

但是：

仍然屬於：

應稅交易。

只是：

稅率：

0%。

例如：

出口美國。

出口日本。

出口越南。

通常都是：

零稅率。

---

免稅（Exempt）

表示：

法律規定：

不用課稅。

例如：

某些教育、

醫療、

金融服務。

ERP：

通常：

Zero Rated

跟：

Exempt

會分開設定。

---

# 二、外銷為什麼通常使用 Commercial Invoice？

出口時：

國外客戶：

不需要：

台灣統一發票。

他們需要的是：

```
Commercial Invoice
```

內容通常包括：

- 客戶名稱
- 商品名稱
- 數量
- 單價
- 總金額
- 幣別
- Incoterms
- 出口資訊

因此：

ERP：

出口流程：

通常是：

```
Sales Order

↓

Delivery

↓

Commercial Invoice

↓

出口報關
```

而不是：

開立：

台灣統一發票。

---

# 三、什麼是人工選號？

正常情況：

ERP：

開立統一發票時：

都是：

依號碼：

自動遞增。

例如：

```
AA-00000001

↓

AA-00000002

↓

AA-00000003
```

ERP：

自動：

依序取號。

這是：

最常見方式。

---

# 四、為什麼需要人工選號？

某些公司：

會預留：

一些發票號碼。

例如：

```
AA-00000150

~

AA-00000180
```

暫時：

不使用。

保留下來。

原因：

可能是：

客戶要求。

---

例如：

今天：

```
6/3

出貨。
```

但是：

客戶希望：

```
5/31

開立發票。
```

公司：

可能：

之前：

就保留：

5/31：

最後幾張發票。

之後：

由財務：

人工指定：

```
AA-00000179
```

因此：

ERP：

提供：

```
人工選號
```

功能。

---

# 五、手寫發票是什麼？

早期：

很多公司：

使用：

政府印製：

統一發票。

例如：

```
AA-12345678
```

直接：

人工填寫。

ERP：

之後：

需要：

把：

這張：

已經寫好的發票。

輸入：

系統。

因此：

ERP：

提供：

```
手寫發票
```

模式。

主要用途：

- 登錄發票號碼
- 記錄金額
- 保持帳務一致

ERP：

通常：

會管理：

發票號段。

```

00001000
```

系統：

哪些：

哪些：

# 七、ERP 發票開立方式

一般有：

### 1、自動取號

ERP：

依序：

開立。

```
AA000101

↓

AA000102

↓

AA000103
```

適用：

大部分公司。

---

### 2、人工選號

財務：

自行指定：

```
AA000188
```

通常：

因特殊需求：

或補開發票。

---

### 3、手寫發票

ERP：

僅記錄：

已經人工開立的發票。

不負責：

印製。

---

# 八、ERP 為什麼要提供這些功能？

因為：

ERP：

除了管理：

商業流程。

還必須：

符合：

各國法規。

例如：

台灣：

統一發票制度。

ERP：

因此：

需要：

- 管理發票號段
- 管理作廢發票
- 管理人工選號
- 管理補開發票
- 管理電子發票
- 管理紙本發票

---

# 九、學習重點

✅ 外銷通常適用零稅率（Zero Rated），不是免稅（Exempt）。

✅ 外銷客戶通常使用 Commercial Invoice 作為請款與出口文件。

✅ 台灣統一發票主要適用於國內交易，需符合相關法規。

✅ ERP 的發票號碼通常由系統依序管理，也支援人工選號。

✅ 手寫發票模式主要是為了將紙本發票資料補登到 ERP。

---

# 十、ERP 一句話總結

> **商業發票（Commercial Invoice）解決的是「客戶付款」；統一發票／稅務發票（Tax Invoice）解決的是「政府稅務管理」。**

ERP 必須同時支援：

- 商業交易
- 稅務法規
- 發票號碼管理

才能符合企業的實際作業需求。
