# 視覺定位和CAD 應用

> 分類：未分類  
> Tags：無  
> 建立日期：2026-07-22

* ============================================================
* HALCON 模擬範例
*
* 功能：
* 1. 使用九點標定，建立「影像 Pixel → Robot mm」轉換
* 2. 相機找到產品的兩個定位孔 A、B
* 3. 將定位孔從 Pixel 轉成 Robot 座標
* 4. 使用 CAD 中的定位孔 A、B 對齊實際產品
* 5. 將 CAD 中所有螺絲孔一次轉換成 Robot 座標
*
* 座標系：
* Image Coordinate : Pixel
* CAD Coordinate   : mm
* Robot Coordinate : mm
* ============================================================


* ------------------------------------------------------------
* 第一部分：建立九點標定資料
* ------------------------------------------------------------
*
* 假設我們讓機械手走到九個已知位置，
* 然後記錄每個 Robot 位置在相機影像中的 Row、Column。
*
* 這就是你以前做的九點標定。
*
* 注意：
* HALCON 在處理影像座標時，通常使用：
* Row    = 影像垂直方向
* Column = 影像水平方向
*
* 但是 affine_trans_point_2d 的參數名稱叫 Px、Py。
* 這裡統一規定：
*
* Px = Row
* Py = Column
*
* 只要從頭到尾保持一致就可以。
* ------------------------------------------------------------

ImageRowsCalib := [200, 200, 200, \
                   500, 500, 500, \
                   800, 800, 800]

ImageColsCalib := [300, 600, 900, \
                   300, 600, 900, \
                   300, 600, 900]


* 對應的機械手實際位置，單位 mm
*
* 這裡假設：
* 影像 Row 增加，Robot X 增加
* 影像 Column 增加，Robot Y 增加
*
* 真正機台上的方向可能相反，
* 但標定矩陣會自動計算方向與比例。

RobotXCalib := [100, 100, 100, \
                200, 200, 200, \
                300, 300, 300]

RobotYCalib := [50, 150, 250, \
                50, 150, 250, \
                50, 150, 250]


* ------------------------------------------------------------
* 第二部分：計算 Pixel → Robot 的仿射轉換矩陣
* ------------------------------------------------------------
*
* vector_to_hom_mat2d 使用多組對應點，
* 計算一個 2D 仿射轉換矩陣。
*
* 輸入：
* ImageRowsCalib、ImageColsCalib = 相機影像座標
*
* 輸出：
* RobotXCalib、RobotYCalib       = 機械手座標
*
* 得到：
* HomMat2DPixelToRobot
*
* 它代表：
*
* [Image Row, Image Column]
*              ↓
* [Robot X, Robot Y]
* ------------------------------------------------------------

vector_to_hom_mat2d (ImageRowsCalib, ImageColsCalib, \
                     RobotXCalib, RobotYCalib, \
                     HomMat2DPixelToRobot)


* ------------------------------------------------------------
* 第三部分：定義 CAD 裡面的理論孔位
* ------------------------------------------------------------
*
* 假設 DXF/CAD 裡有六個螺絲孔。
*
* CAD 單位為 mm。
*
* 這些座標是相對於產品 CAD 原點的理論位置。
* ------------------------------------------------------------

CadScrewX := [20.0, 60.0, 100.0, \
              20.0, 60.0, 100.0]

CadScrewY := [20.0, 20.0, 20.0, \
              70.0, 70.0, 70.0]


* CAD 裡面另外定義兩個定位孔：
*
* 定位孔 A 是產品的基準原點
* 定位孔 B 用來決定產品方向
*
* CAD 中：
* A = (0, 0)
* B = (120, 0)
*
* 因為 A → B 是水平方向，
* 所以 CAD 產品的原始角度是 0 度。

CadDatumAX := 0.0
CadDatumAY := 0.0

CadDatumBX := 120.0
CadDatumBY := 0.0


* ------------------------------------------------------------
* 第四部分：模擬相機找到實際產品的定位孔
* ------------------------------------------------------------
*
* 實際運行時，這些 Pixel 座標應該來自：
*
* threshold
* connection
* select_shape
* smallest_circle
*
* 或 shape-based matching。
*
* 這裡先直接使用模擬值。
*
* 假設產品送進來後：
* - 有平移
* - 有旋轉
*
* 所以定位孔 A、B 在影像上的位置每次都不一樣。
* ------------------------------------------------------------

ImageRowDatumA := 430.0
ImageColDatumA := 470.0

ImageRowDatumB := 485.0
ImageColDatumB := 825.0


* ------------------------------------------------------------
* 第五部分：定位孔 Pixel → Robot
* ------------------------------------------------------------
*
* 使用前面九點標定得到的矩陣，
* 把相機找到的定位孔座標轉成 Robot mm。
* ------------------------------------------------------------

affine_trans_point_2d (HomMat2DPixelToRobot, \
                       ImageRowDatumA, ImageColDatumA, \
                       RobotDatumAX, RobotDatumAY)

affine_trans_point_2d (HomMat2DPixelToRobot, \
                       ImageRowDatumB, ImageColDatumB, \
                       RobotDatumBX, RobotDatumBY)


* 現在已經得到：
*
* RobotDatumA = 實際產品定位孔 A 的 Robot 座標
* RobotDatumB = 實際產品定位孔 B 的 Robot 座標


* ------------------------------------------------------------
* 第六部分：計算 CAD 中產品的方向
* ------------------------------------------------------------
*
* atan2(ΔY, ΔX) 計算 A → B 的方向角。
*
* CAD 原始角度通常是 0，
* 但不要直接寫死，計算會比較安全。
* ------------------------------------------------------------

CadDeltaX := CadDatumBX - CadDatumAX
CadDeltaY := CadDatumBY - CadDatumAY

CadAngle := atan2(CadDeltaY, CadDeltaX)


* ------------------------------------------------------------
* 第七部分：計算實際產品在 Robot 座標中的方向
* ------------------------------------------------------------
*
* 由實際定位孔 A、B 計算產品現在旋轉了多少。
* ------------------------------------------------------------

RobotDeltaX := RobotDatumBX - RobotDatumAX
RobotDeltaY := RobotDatumBY - RobotDatumAY

RobotAngle := atan2(RobotDeltaY, RobotDeltaX)


* 角度差就是產品相對 CAD 旋轉的角度
ProductRotation := RobotAngle - CadAngle


* 將弧度轉成角度，方便人員查看
ProductRotationDegree := deg(ProductRotation)


* ------------------------------------------------------------
* 第八部分：建立 CAD → Robot 剛體轉換
* ------------------------------------------------------------
*
* vector_angle_to_rigid 使用：
*
* CAD 定位孔 A 的位置與角度
*                ↓
* 實際 Robot 定位孔 A 的位置與角度
*
* 建立一個只有：
*
* 1. Rotation 旋轉
* 2. Translation 平移
*
* 的剛體轉換。
*
* 沒有縮放。
*
* 這非常適合 CAD 導向定位，因為產品的實際尺寸
* 原則上不應該因為相機拍照而改變。
* ------------------------------------------------------------

vector_angle_to_rigid (CadDatumAX, CadDatumAY, CadAngle, \
                       RobotDatumAX, RobotDatumAY, RobotAngle, \
                       HomMat2DCadToRobot)


* ------------------------------------------------------------
* 第九部分：整張 CAD 的螺絲孔一次轉到 Robot
* ------------------------------------------------------------
*
* 不需要一顆一顆教導 Robot。
*
* 只要把所有 CAD 螺絲孔座標放進去，
* HALCON 就會一次算出全部 Robot 座標。
* ------------------------------------------------------------

affine_trans_point_2d (HomMat2DCadToRobot, \
                       CadScrewX, CadScrewY, \
                       RobotScrewX, RobotScrewY)


* ------------------------------------------------------------
* 第十部分：輸出結果
* ------------------------------------------------------------

dev_clear_window ()

disp_message (WindowHandle, \
              'CAD 導向螺絲定位模擬', \
              'window', 20, 20, 'black', 'true')


disp_message (WindowHandle, \
              '產品旋轉角度：' + ProductRotationDegree$'.3f' + ' degree', \
              'window', 60, 20, 'black', 'true')


disp_message (WindowHandle, \
              '定位孔 A Robot：X=' + RobotDatumAX$'.3f' + \
              ', Y=' + RobotDatumAY$'.3f', \
              'window', 100, 20, 'black', 'true')


disp_message (WindowHandle, \
              '定位孔 B Robot：X=' + RobotDatumBX$'.3f' + \
              ', Y=' + RobotDatumBY$'.3f', \
              'window', 140, 20, 'black', 'true')


* ------------------------------------------------------------
* 顯示六顆螺絲最終的 Robot 座標
* ------------------------------------------------------------

for Index := 0 to |RobotScrewX| - 1 by 1

    ScrewNumber := Index + 1

    Message := '螺絲 ' + ScrewNumber + \
               '：Robot X=' + RobotScrewX[Index]$'.3f' + \
               ', Y=' + RobotScrewY[Index]$'.3f'

    disp_message (WindowHandle, Message, \
                  'window', 180 + Index * 35, 20, \
                  'black', 'true')

endfor


* ------------------------------------------------------------
* 模擬 Robot 鎖螺絲
* ------------------------------------------------------------
*
* 真正專案裡，這裡通常會把 RobotScrewX、RobotScrewY
* 透過 TCP/IP、Socket、PLC 或 Fieldbus 傳給機械手。
*
* Z、Rz、鎖付高度、扭力和鎖付順序，
* 通常由 Robot Recipe 或產品配方管理。
* ------------------------------------------------------------

for Index := 0 to |RobotScrewX| - 1 by 1

    TargetRobotX := RobotScrewX[Index]
    TargetRobotY := RobotScrewY[Index]

    * 模擬固定的鎖付高度
    TargetRobotZ := 15.0

    * 模擬工具方向
    TargetRobotRZ := ProductRotationDegree

    * 實際應用範例：
    *
    * send_tuple(SocketHandle, TargetRobotX)
    * send_tuple(SocketHandle, TargetRobotY)
    * send_tuple(SocketHandle, TargetRobotZ)
    * send_tuple(SocketHandle, TargetRobotRZ)
    *
    * 然後等待 Robot 回覆鎖付完成。

endfor
