# 預開發票自動轉成銷貨訂單？ 這又是什麼流程？

> 分類：ERP  
> 學習日期：2026-07-13  
> 狀態：持續複習、已完成

## 問題

預開發票自動轉成銷貨訂單？ 這又是什麼流程？

## 學習整理

# ERP 筆記：預開發票 (Invoice Draft) 與銷貨訂單 (Sales Order) 的差異

> 適合 NotebookLM 學習版本
>
> 主題：為什麼 ERP 有預開發票？它和銷貨訂單有什麼不同？

---

# 一、很多人第一次看到都會疑惑

ERP 裡常看到：

```
報價單
    ↓
預開發票
    ↓
銷貨訂單
```

第一個疑問通常是：

> 「預開發票不是跟訂單很像嗎？」

答案是：

**是的，資料幾乎一樣。**

真正不同的是：

**商業意義（Business Meaning）不同。**

---

# 二、三種單據比較

| 項目 | 報價單 (Quotation) | 預開發票 (Invoice Draft) | 銷貨訂單 (Sales Order) |
|------|-------------------|--------------------------|-------------------------|
| 客戶 | ✔ | ✔ | ✔ |
| 品號 | ✔ | ✔ | ✔ |
| 品名 | ✔ | ✔ | ✔ |
| 數量 | ✔ | ✔ | ✔ |
| 單價 | ✔ | ✔ | ✔ |
| 金額 | ✔ | ✔ | ✔ |
| 幣別 | ✔ | ✔ | ✔ |
| 稅率 | 可有可無 | ✔ | 可有可無 |
| 備註 | ✔ | ✔ | ✔ |

可以發現：

**90% 以上的欄位都相同。**

---

# 三、真正不同的是「商業意義」

## 1. 報價單 (Quotation)

代表：

> 「這是我的報價，你要不要購買？」

例如：

```
腳踏車
100 台
單價：10,000 元
```

此時：

客戶還沒有正式下訂。

---

## 2. 預開發票 (Invoice Draft)

代表：

> 「如果交易成立，就依照這份內容開正式發票。」

通常代表：

- 等待付款
- 等待財務確認
- 等待主管審核
- 等待正式開票

它不是正式發票。

通常具有：

- 可以修改
- 可以刪除
- 不會過帳
- 不會影響會計
- 不會產生稅務

---

## 3. 銷貨訂單 (Sales Order)

代表：

> 「公司正式接受客戶的訂單。」

當 ERP 建立 Sales Order 後，

很多後續流程就會開始。

例如：

```
Sales Order

↓

MRP

↓

採購

↓

生產排程

↓

製造工令

↓

MES

↓

出貨
```

因此：

Sales Order 通常代表：

正式接單。

---

# 四、ERP 為什麼三張單據資料都差不多？

因為：

它們描述的是：

**同一筆交易。**

例如：

```
客戶：ABC

產品：BIKE-001

數量：100

單價：10,000
```

不論是：

- 報價
- 預開發票
- 銷貨訂單

資料幾乎都相同。

ERP 不希望使用者：

重新輸入一次。

所以：

都是利用：

**轉單（Document Conversion）**

完成。

---

# 五、ERP 的轉單概念

ERP 最重要的觀念之一就是：

**上一張單據，轉成下一張單據。**

例如：

```
Quotation

↓

Sales Order

↓

Delivery

↓

Invoice
```

或：

```
Quotation

↓

Invoice Draft

↓

Sales Order

↓

Delivery

↓

Invoice
```

ERP 不需要重新輸入資料。

而是：

直接 Copy。

---

# 六、真正改變的是什麼？

其實：

不是資料。

而是：

**文件狀態（Status）**

例如：

同一筆資料：

```
ABC 公司

BIKE-001

100 台
```

如果：

```
Status = Quotation
```

表示：

只是報價。

如果：

```
Status = Invoice Draft
```

表示：

等待付款。

如果：

```
Status = Sales Order
```

表示：

正式接單。

因此：

ERP 只是：

改變文件代表的商業意義。

---

# 七、ERP 資料表通常長什麼樣？

例如：

Quotation

```
QuotationNo

Customer

ItemCode

Qty

Price
```

Invoice Draft

```
DraftNo

Customer

ItemCode

Qty

Price
```

Sales Order

```
SONo

Customer

ItemCode

Qty

Price
```

欄位幾乎相同。

因此：

ERP 很容易：

直接轉單。

---

# 八、為什麼 ERP 一直在 Copy？

如果沒有轉單：

業務需要：

```
報價輸入一次

↓

訂單再輸入一次

↓

發票再輸入一次
```

容易發生：

- 品號輸入錯誤
- 單價不同
- 數量錯誤
- 折扣錯誤

ERP：

直接 Copy。

只修改：

- Document Type
- Status
- 單號

即可。

---

來源：

Quotation 或 Invoice Draft。

---

# 十、哪些公司會使用 Invoice Draft？

通常：

- 客製化設備
- 接案公司
- 預收款公司
- 補習班
- 教育訓練
- 活動報名
- 會員制
- 預購商品

因為：

付款後，

才正式建立 Sales Order。

---

---

# 十二、學習重點

✅ 報價單、預開發票、銷貨訂單，資料內容非常相似。

✅ 最大差異不是資料，而是每張單據代表的商業意義。

✅ ERP 的核心設計理念是「轉單（Document Conversion）」。

✅ 每張新單據，大部分資料都是由上一張單據複製而來。

✅ ERP 的目的不是一直建立新資料，而是讓同一筆交易，在不同階段以不同單據流轉，同時保留完整的追蹤關係。

---

# ERP 核心觀念（最重要）

ERP 並不是一直新增新的交易。

而是：

同一筆商業交易，

透過不同文件：

```
Quotation
    ↓
Invoice Draft（可選）
    ↓
Sales Order
    ↓
Delivery
    ↓
Invoice
    ↓
Accounts Receivable
    ↓
Receipt
```

每一張單據都代表：

**同一筆交易在不同商業階段的狀態。**

這就是 ERP 「單據流（Document Flow）」的核心設計思想。
