# 銷貨訂單自動轉成出貨單 ， 出貨單自動產生發票

> 分類：ERP  
> 學習日期：2026-07-12  
> 狀態：持續複習、已完成

## 問題

銷貨訂單自動轉成出貨單 ， 出貨單自動產生發票

## 學習整理

# ERP 筆記：銷貨訂單 → 出貨單 → 發票 為什麼要自動轉？

> 適合 NotebookLM 學習版本
>
> 主題：ERP 最重要的銷售資料流

---

# 一、ERP 最標準的銷售流程

製造業 ERP 最常見的流程如下：

```text
客戶下訂單
      ↓
銷貨訂單 (Sales Order)
      ↓
出貨單 (Delivery)
      ↓
發票 (Invoice)
      ↓
應收帳款 (Accounts Receivable)
      ↓
收款 (Receipt)
```

ERP 幾乎都是利用：

**上一張單據轉下一張單據。**

而不是重新輸入。

---

# 二、為什麼要先建立銷貨訂單？

例如：

ABC 公司訂購：

```
BIKE-001

100 台
```

ERP 建立：

```
Sales Order

SO000123

客戶：ABC

產品：BIKE-001

數量：100
```

Sales Order 代表：

> 公司正式接受客戶訂單。

但是：

這時候：

還沒有出貨。

因此：

庫存不能扣。

---

# 三、什麼是出貨單？

當倉庫開始出貨：

ERP 建立：

```
Delivery

DO000456

來源：

SO000123
```

表示：

產品真的離開倉庫。

例如：

```
100 台

今天出貨
```

ERP 這時候會：

- 扣成品庫存
- 紀錄出貨日期
- 紀錄物流資訊
- 更新訂單出貨數量

因此：

Delivery 的重點：

**物流。**

---

# 四、為什麼出貨單不能直接叫發票？

因為：

物流：

跟：

財務。

是兩件事情。

例如：

今天：

100 台出貨。

但是：

發票：

可能：

明天才開。

甚至：

月底一起開。

因此：

ERP 必須：

把：

物流。

跟：

會計。

分開。

---

# 五、發票是什麼？

發票代表：

> 公司正式向客戶請款。

例如：

```
Invoice

INV000789

來源：

DO000456
```

ERP：

建立發票後：

通常會：

```
借：

應收帳款

貸：

銷貨收入

貸：

銷項稅額
```

因此：

Invoice：

開始影響：

會計。

---

# 六、ERP 為什麼要自動轉？

假設：

沒有轉單。

業務：

輸入：

```
Sales Order

↓

Delivery

↓

Invoice
```

三次。

容易發生：

- 品號不同
- 數量不同
- 單價錯誤
- 客戶錯誤

ERP：

改成：

```
Sales Order

↓

Copy

↓

Delivery

↓

Copy

↓

Invoice
```

不用重新輸入。

---

# 七、ERP 真正改變的是什麼？

其實：

資料：

幾乎相同。

例如：

```
Customer

ABC
```

```
BIKE-001

100

10,000
```

一直都相同。

ERP：

只是：

改變：

文件狀態。

---

Sales Order

代表：

```
客戶下訂
```

---

Delivery

代表：

```
商品已出貨
```

---

Invoice

代表：

```
開始請款
```

---

# 八、ERP 資料流

```
Sales Order

SO000123

        │
        │ Copy
        ▼

Delivery

DO000456

        │
        │ Copy
        ▼

Invoice

INV000789

        │
        ▼

Accounts Receivable

AR000001

        │
        ▼

Receipt

RC000001
```

每張單據都保留：

來源單號。

因此：

Invoice 可以知道：

來源：

Delivery。

Delivery 又知道：

來源：

Sales Order。

形成完整追蹤。

---

# 九、腳踏車工廠範例

客戶：

ABC

訂購：

```
BIKE-001

100 台
```

建立：

```
Sales Order

SO10001
```

↓

倉庫：

今天出貨：

100 台。

建立：

```
Delivery

DO20001
```

↓

財務：

開立：

```
Invoice

INV30001
```

↓

ERP：

產生：

```
借：

應收帳款

貸：

銷貨收入

貸：

銷項稅額
```

↓

等待：

收款。

---

# 十、為什麼不能直接 Sales Order → Invoice？

因為：

很多情況：

**實際出貨與訂單不一致。**

例如：

訂單：

```
100 台
```

今天：

只有：

```
60 台
```

ERP：

建立：

```
Delivery

60 台
```

剩下：

```
40 台
```

下週：

再出。

因此：

ERP 可以：

```
Sales Order

100

↓

Delivery

60

↓

Delivery

40
```

最後：

可以：

```
60 台

+

40 台

↓

一張發票
```

或：

```
兩張發票
```

依公司規則處理。

所以：

Delivery：

不能省略。

---

# 十一、ERP 核心設計理念

ERP 並不是一直建立新資料。

而是：

同一筆交易。

一路往下流轉。

```
Quotation
      ↓
Sales Order
      ↓
Delivery
      ↓
Invoice
      ↓
Accounts Receivable
      ↓
Receipt
      ↓
General Ledger
```

每張單據：

資料幾乎相同。

真正改變的是：

- 商業意義
- 文件狀態
- 對公司的影響

---

# 十二、學習重點

✅ Sales Order：正式接受客戶訂單。

✅ Delivery：商品實際出貨，影響庫存。

✅ Invoice：正式向客戶請款，影響會計。

✅ ERP 利用轉單避免重複輸入。

✅ 每張單據都保留來源單號，可完整追蹤交易歷程。

✅ Delivery 是物流文件；Invoice 是財務文件，兩者不可混為一談。

---

# ERP 一句話總結

> **Sales Order 管「客戶要什麼」，Delivery 管「實際出了什麼」，Invoice 管「應該向客戶收多少錢」。**

因此，ERP 將這三種不同職責拆成不同單據，再透過「轉單」串成完整的資料流。
